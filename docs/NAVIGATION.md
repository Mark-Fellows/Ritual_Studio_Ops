# Ritual Studio Ops - Navigation Map

**Date:** 2026-05-24
**Purpose:** show how the portals, dashboards and external platforms link to each other, and how to tell the new merged system from the legacy ones at a glance.

For the file that backs each screen, see `SOURCE_OF_TRUTH.md`.

---

## The rule

- **Emblem badge top-left of the header** => you are in the new (merged) system. A change goes into a file under `Ritual_Studio_Ops/app/`.
- **No badge** => a legacy system, retiring at Phase 6 cutover. Do not change.
- **URL test:** `ritual-studio-ops.pages.dev` is new. `ritual-teacher-management.pages.dev` and `ritual-cover-management.pages.dev` are legacy.

## Notes on the diagram

- **The portal's two "internal app" cards (Cover Dashboard and Teacher Portal) both open the same merged app.** They preserve the muscle memory of the two old entry points but resolve to one file (`app/ritual-studio-ops-v2.html`). The merged app then chooses Teachers or Cover from its own nav.
- **Studio Timetable** has not been merged. It is served from the legacy Cover Management site and the portal links there directly. It does not carry the emblem badge. Folding it into Studio Ops is part of the planned Phase 6 work below.
- **Ritual Dashboard** and **Campaign Planning** are separate Cloudflare Pages projects, not part of the merger.
- **Legacy URLs still respond.** Bookmarks to `ritual-teacher-management.pages.dev` and `ritual-cover-management.pages.dev` continue to work during the parallel run. They are not linked from the new portal. They retire at Phase 6.
- **Backend is shared.** Every internal box above reads and writes the same Supabase project. A database change is felt everywhere.

## Planned reduction to 3 Pages projects (decision 2026-05-24)

At Phase 6 cutover:
- Retire `ritual-teacher-management.pages.dev` (project 2).
- Retire `ritual-cover-management.pages.dev` (project 3), having first moved Studio Timetable into Studio Ops. The timetable HTML is generated by `services/cover/tools/generate_timetable_html.py` - point its output at `Ritual_Studio_Ops/app/studio_timetable.html` and update the portal card to link to `./studio_timetable.html`.
- Keep Ritual Dashboard (project 4) and Campaign Planning (project 5) as separate Pages projects for now. The Management Portal already routes to them in one click.

End state at Phase 6: **three Pages projects** - Studio Ops, Dashboard, Campaign Planning.

Going further (three to one - folding Dashboard and Campaign Planning into Studio Ops as sub-paths) is technically feasible but couples release cadences and crosses ownership boundaries. Revisit only after Phase 6 has been stable for a sustained period.

## Prerequisite to any consolidation

The canonical RSO Git repository currently has no GitHub remote (the audit's H4). Whatever consolidation is done at Phase 6, that remote needs to be sorted first so Git-integration deploys to Cloudflare Pages work cleanly.
